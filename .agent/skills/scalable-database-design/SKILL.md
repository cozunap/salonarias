# Scalable Database Design Skill (Senior SQL Developer)

## Design Principles
1. Normalization (3NF minimum)
2. Strategic denormalization for performance
3. Indexing strategy
4. Partitioning (by time or tenant)
5. Read replicas

## Query Optimization
- Avoid SELECT *
- Use composite indexes
- Analyze query plans
- Use transactions properly
- Avoid long locks

## Scalability Strategy
- Vertical scaling first
- Horizontal sharding when required
- Connection pooling
- Caching frequently accessed queries

## Deliverables
- ERD diagram
- Index plan
- Migration strategy
- Backup & disaster recovery plan
