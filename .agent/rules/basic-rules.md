GOOGLE ANTIGRAVITY

Professional Operational Rules – Enterprise Edition

⸻

1. Foundational Principle

AntiGravity is not a project. It is an execution operating system.

Core rule:

If it is not documented, it does not exist.
If it is not measured, it does not improve.
If it is not validated, it is not deployed.

⸻

2. Universal Development Cycle (Mandatory)

Every task follows this flow without exception:
	1.	Diagnosis
	•	Define the exact problem
	•	Identify technical and business impact
	•	Assess risks
	2.	Design
	•	Clear architecture
	•	Scalability validated
	•	Security considered
	3.	Implementation
	•	Clean code
	•	Strict modularity
	•	Professional naming conventions
	4.	Validation
	•	Automated testing
	•	Structural review
	•	Technical audit
	5.	Documentation
	•	Update .md files
	•	Update diagrams
	•	Update technical decisions (ADR)
	6.	Re-Verification
	•	Internal score ≥ 9
	•	No repeated errors
	•	Performance validated

⸻

3. Internal Scoring System

Each module must be evaluated across:
	•	Architecture
	•	Security
	•	Performance
	•	Scalability
	•	UX/UI
	•	SEO (if applicable)
	•	Documentation
	•	Test Coverage

Golden Rule
	•	Score ≥ 9 → Approved
	•	Score 8 → Optimize before merge
	•	Score ≤ 7 → Rebuild
	•	3 failed attempts → Escalate to senior review

⸻

4. Global Auto-Correction Protocol

Absolute rule:

Never fail twice for the same root cause.

Mandatory cycle:
	1.	Detect root cause
	2.	Patch the solution
	3.	Update technical memory/documentation
	4.	Add a preventive test
	5.	Re-audit

Documented memory is as important as code.

⸻

5. Code Standards

Naming Conventions
	•	Variables → camelCase
	•	Components → PascalCase
	•	Files → kebab-case
	•	Constants → UPPER_SNAKE_CASE

Core Principles
	•	DRY
	•	SOLID
	•	KISS
	•	Clean Architecture
	•	Separation of Concerns

Strictly Prohibited
	•	Duplicate code
	•	Business logic inside UI components
	•	Unindexed database queries
	•	Ambiguous variable names
	•	Meaningless comments

⸻

6. AntiGravity Base Architecture

Mandatory:
	•	Decoupled frontend
	•	Modular backend
	•	Normalized database
	•	Versioned API
	•	Structured logging
	•	Active monitoring

Enterprise-grade requires:
	•	CDN
	•	Caching layer
	•	Rate limiting
	•	Observability
	•	Automated CI/CD

⸻

7. Non-Negotiable Security

Always enforce:
	•	Frontend and backend validation
	•	Input sanitization
	•	Protection against XSS / SQL injection
	•	Secure JWT or session handling
	•	Environment variables for sensitive data
	•	Role-based access control

⸻

8. Performance First

Every feature must:
	•	Load in < 2 seconds
	•	Use lazy loading where applicable
	•	Optimize database queries
	•	Minimize re-renders
	•	Compress and optimize assets

Performance is not a final optimization.
It is an architectural decision.

⸻

9. Professional Commit Protocol

Mandatory format:
[ANTIGRAVITY] Type: Clear description

Examples:
[ANTIGRAVITY] Feature: Add scalable product filtering system
[ANTIGRAVITY] Fix: Resolve race condition in auth middleware

When score ≥ 9:
[AUTO-HEALED]

10. Global Scalability Rules

The system must support:
	•	10x traffic without re-architecture
	•	Multi-tenant readiness
	•	Internationalization prepared
	•	Cloud-native deployment
	•	Database scaling strategy

Never build for today.
Build for 5 years ahead.

⸻

11. Final AntiGravity Principles
	•	Clarity > Complexity
	•	UX > Technical ego
	•	Architecture > Trends
	•	Systems > Hacks
	•	Automation > Manual work
	•	Documentation is part of development
	•	If it is not written, it does not exist

⸻

12. AntiGravity Mode Activated

These rules are not suggestions.
They are the operational standard.

AntiGravity does not improvise.
AntiGravity executes.
